\#\# How to Use This Repo with Grok (or any AI)

\#\#\# Loading the Canon  
In a new conversation, start with:

\> \*\*"Load Mercy Shards Canon from GitHub"\*\*    
\> Repo: \`tryaiforwealth-blip/mercy-shards-novel\` (or your username/repo)    
\> Branch: \`main\`

Grok will then read the key files from the \`canon/\` folder and confirm it is operating inside the established universe.

\#\#\# Updating Documents  
\- When asking Grok to revise something, be specific:  
  \- “Update technology-codex.md with new nanobot details”  
  \- “Write Chapter 4 and save as ch04-....md”  
\- After Grok produces updated content, copy the output and commit it.

\#\#\# Recommendations for Grok  
\- Always prioritize files in this order: \`canon/\` → \`outline.md\` → current chapter drafts.  
\- When suggesting changes, create a clear diff or full revised file.  
\- Maintain mystery-first tone in early chapters.  
\- Use real Tennessee geography where appropriate (Allens Market, Blackburn Fork, Cummins Falls, Flynn Creek watershed, etc.).

\#\#\# Branching Suggestion  
\- \`main\` — Stable canon \+ latest approved chapters  
\- \`draft\` — Work-in-progress chapters (merge to main when ready)  
\- \`experiment\` — Wild ideas, alternate scenes

This structure keeps everything organized and makes it easy for Grok (or any AI) to stay consistent across sessions.
